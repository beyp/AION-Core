# Roadmap

## Short term
- Keep the repository documentation current and agent-friendly.
- Expand regression coverage around routing, discovery, and startup behavior.
- Preserve Windows and VS Code compatibility while making changes.

## Medium term
- Improve test coverage for API routes, store operations, and updater logic.
- Reduce startup fragility by isolating optional integrations.
- Make the configuration flow clearer for new contributors and agents.

## Long term
- Add more automated checks for runtime safety and configuration errors.
- Improve observability for app launch, backup, and update flows.
- Continue modularizing connectors so new integrations can be added with minimal risk.
